CSS Box Model: content-box vs. border-box

This page compares the default browser behavior (content-box) against the modern box-sizing fix (border-box).

1. Default Sizing (content-box)

Here, the set dimensions apply only to the content area. Padding and borders are added on top, inflating the total size.

Content Area (300px × 100px)

Dimensions Math:

  • Declared Width: 300px
  • Padding (Left/Right): 20px + 20px = 40px
  • Border (Left/Right): 5px + 5px = 10px
  • Total Visible Width: 300 + 40 + 10 = 350px
  • Declared Height: 100px
  • Padding (Top/Bottom): 20px + 20px = 40px
  • Border (Top/Bottom): 5px + 5px = 10px
  • Total Visible Height: 100 + 40 + 10 = 150px

2. Standard Sizing (border-box)

Here, the padding and border are included inside the declared width and height. The content area shrinks to accommodate them.

Content Area (250px × 50px)

Dimensions Math:

  • Declared Width: 300px
  • Padding (Left/Right): 20px + 20px = 40px (internalized)
  • Border (Left/Right): 5px + 5px = 10px (internalized)
  • Total Visible Width: 300px (exactly matches declared)
  • Inner Content Width: 300 - 40 - 10 = 250px
  • Declared Height: 100px
  • Padding (Top/Bottom): 20px + 20px = 40px (internalized)
  • Border (Top/Bottom): 5px + 5px = 10px (internalized)
  • Total Visible Height: 100px (exactly matches declared)
  • Inner Content Height: 100 - 40 - 10 = 50px
