1. Sticky Position (position: sticky)

Scroll down inside this container or the page. The sub-header bar above is positioned sticky. It behaves like a normal relative element until you scroll past it, then it locks (sticks) to the top of the screen or container viewport.

This is commonly used for persistent menus, table headers, or sidebar summaries.

2. Static Position (position: static)

This is the default positioning method for all elements. The element flows naturally according to the document layout. Setting offsets (top, left, etc.) has no effect on static elements.

I am Static (Default Flow)

3. Relative Position (position: relative)

The element is offset relative to its normal position in the document flow. The surrounding elements act as if the element was never moved, meaning its original space is fully preserved.

4. Absolute Position (position: absolute)

The element is removed from the normal layout flow and positioned precisely relative to its nearest positioned (non-static) parent. In this case, the red "NEW" badge in the top-right corner is positioned absolutely inside this relative section card.

If no positioned parent existed, it would align relative to the HTML body itself.

5. Fixed Position (position: fixed)

Look at the bottom-right corner of your browser screen. There is a blue floating "Fixed Feedback" badge. It is positioned using position: fixed.

A fixed element is removed from the layout flow and positioned relative to the browser viewport itself. It remains locked in place even when you scroll the webpage.
